drift-slider-react v0.1.0

2026-06-26 — Official React wrapper

New Package

新套件

  • drift-slider-react — an official thin React wrapper, installed alongside the core: npm i drift-slider drift-slider-react.
  • drift-slider-react — 官方輕量 React 包裝，與核心一起安裝：npm i drift-slider drift-slider-react。
  • <DriftSlider> component — renders the slider scaffold and accepts options, modules, an on event map, and shortcut event props (onSlideChange, onReachEnd, …).
  • <DriftSlider> 元件 — 自動產生 slider 結構，接受 options、modules、on 事件物件與快捷事件 props（onSlideChange、onReachEnd…）。
  • useDriftSlider hook & <Slide> helper — a headless hook for full control, plus a <Slide> wrapper for list items.
  • useDriftSlider hook 與 <Slide> helper — 提供完全掌控的 headless hook，以及包裝 list item 的 <Slide>。
  • Imperative ref handle — the ref exposes slideTo, slideNext, slidePrev, update, and the live instance.
  • 命令式 ref handle — ref 提供 slideTo、slideNext、slidePrev、update 與即時 instance。
  • SSR-safe — ships with the 'use client' banner and initialises only in an effect; supports React 18 & 19. Types are derived from the core package.
  • SSR 安全 — 內建 'use client'、僅在 effect 內初始化；支援 React 18 與 19；型別由核心套件衍生。
  • Auto-update on children change — adding, removing, or reordering slides refreshes the slider automatically.
  • 子節點變更自動更新 — 新增、移除或重排 slide 會自動刷新 slider。

Notes

注意事項

  • Memoize options/modules (or define them outside render) — passing new object/array literals every render re-initialises the slider. Event-handler props may change freely.
  • 請 memoize options/modules（或定義在 render 之外）— 每次 render 都傳入新的物件/陣列字面值會導致 slider 重新初始化；事件處理 props 則可自由變更。
  • Requires drift-slider >= 0.7.0 as a peer dependency.
  • peer dependency 需要 drift-slider >= 0.7.0。

v0.7.0

2026-06-25 — Health Audit & Hardening

New Features

新功能

  • Centered slides — centeredSlides: true now actually centers the active slide with partial peeks of the neighbours (previously a no-op).
  • 置中滑塊 — centeredSlides: true 現在會真正讓活動滑塊置中並露出相鄰滑塊（先前完全無作用）。
  • Autoplay pauses on hidden tab — autoplay now pauses on visibilitychange when the tab is hidden and resumes when visible (skips resume if the pointer is over the slider).
  • 分頁隱藏時暫停自動播放 — 分頁隱藏時依 visibilitychange 暫停，回到可見時恢復（指標懸停在 slider 上時不恢復）。
  • jQuery method calls — $el.driftSlider('slideNext') and other methods now proxy to the instance instead of re-initialising; destroy clears $.data.
  • jQuery 方法呼叫 — $el.driftSlider('slideNext') 等方法現在會代理到實例而非重新初始化；destroy 會清除 $.data。
  • Long swipes — new longSwipes option (default true): a slow drag past longSwipesRatio advances even at low velocity.
  • 長距離滑動 — 新增 longSwipes 選項（預設 true）：即使低速，只要拖曳超過 longSwipesRatio 即換頁。

Bug Fixes

問題修復

  • Grouped navigation — slidesPerGroup ≥ 2 no longer skips groups, and active/visible classes now land on the correct slide.
  • 分組導航 — slidesPerGroup ≥ 2 不再跳過分組，active/visible class 也標在正確的滑塊上。
  • Multi-view end — with slidesPerView > 1, navigating to the end no longer overscrolls past the content into blank space.
  • 多視圖尾端 — slidesPerView > 1 時，導航到底不再過捲露出空白。
  • Event isolation — a throwing event handler no longer aborts the remaining handlers (a buggy callback can't break navigation/pagination/a11y).
  • 事件隔離 — 單一事件處理器拋錯不再中斷其餘處理器（壞掉的 callback 不會連帶弄壞導航/分頁/無障礙）。
  • Accessibility — loop clones are hidden from screen readers with the correct slide count, the full slidesPerView window is exposed, and ARIA / reduced-motion params are restored on destroy.
  • 無障礙 — loop clone 對螢幕報讀隱藏並使用正確張數，完整 slidesPerView 範圍可見，destroy 時還原 ARIA 與 reduced-motion 參數。
  • Keyboard — Home/End target the real first/last slide in loop mode.
  • 鍵盤 — loop 模式下 Home/End 對應真正的首/末張。
  • Navigation & pagination refresh on resize/breakpoint (bullets re-render when the count changes); pagination cleans up its DOM on destroy.
  • 導航與分頁 在 resize/breakpoint 時刷新（總數變動時重繪 bullet）；destroy 時清除 pagination DOM。
  • Breakpoints — toggling loop at a breakpoint now rebuilds/removes the loop clones.
  • 斷點 — 在斷點切換 loop 現在會重建/移除 loop clone。
  • Effects — deck/cards no longer inflate the list width on resize; all effects restore overridden core methods and slidesPerView on destroy; cards loop-jump corrected.
  • 效果 — deck/cards 在 resize 時不再撐爆 list 寬度；所有效果於 destroy 還原被覆寫的核心方法與 slidesPerView；修正 cards 的 loop 跳轉。
  • Thumbs — no duplicate click handlers when re-initialised.
  • 縮圖 — 重新初始化時不再重複綁定 click。
  • Internals — deepMerge hardened against prototype pollution and array aliasing, debounce gains cancel(), and getTranslate parses matrix3d.
  • 內部 — deepMerge 強化防止原型污染與陣列共參，debounce 新增 cancel()，getTranslate 支援 matrix3d。

Notes

注意事項

  • slidesPerGroup > 1 combined with loop falls back to slidesPerGroup: 1 (loop clones are sized per slidesPerView and are not group-aligned).
  • slidesPerGroup > 1 與 loop 併用時會退回 slidesPerGroup: 1（loop clone 依 slidesPerView 配置，無法對齊分組）。

  • Configurable overlay, shadow, active/stack scale, and visible stack count
  • 可設定 overlay、陰影、主卡/牌堆縮放比例及可見牌堆數量
